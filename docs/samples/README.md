# Тестовые JSON для проверки «Лисий диспетчер»

Загрузка: **Добавить → Ручной JSON → Загрузить файл…** или вставить текст → **Применить JSON**.

---

## 1. `01-baseline-mox.json` — эталон ТЗ

**Цель:** сверка со стартовыми данными MOX.

| Проверка | Ожидание |
|----------|----------|
| Записей | 5 |
| Уникальных лис | 4 |
| Лидер | Рыжик (fox_001) ≈ **12.40** |
| Пропуски / предупреждения | нет |

---

## 2. `02-night-shift-extended.json` — расширенная смена

**Цель:** ночной бонус, повторы локации, нормализация окраса (РЫЖЫЙ, черный, серебристый).

| Проверка | Ожидание |
|----------|----------|
| Записей | 8, все приняты |
| Уникальных лис | 5 |
| Лидер | **Ночка (fox_005)** — 3 ночных визита, 2 с добычей, повтор «Туманная тропа» |
| Окраса | все известные (варианты написания приведены к рыжая / черная / серебристая) |
| Окно «Ночь» | при переключении часть дневных obs выпадает из отчёта |

---

## 3. `03-mixed-validation.json` — граничные случаи

**Цель:** обработчик JSON: пропуски, предупреждения, «неизвестен».

| Запись | Что проверяет | Результат |
|--------|---------------|-----------|
| obs_ok01 | валидная база | ✅ принята |
| obs_color_norm | `Рыжий` → рыжая | ✅ + предупреждение о приведении |
| obs_unknown_color | `лазурная` | ✅ окрас **неизвестен**, разноцветная метка |
| obs_bad_location | `Западный бор` | ❌ пропуск — наблюдение не ведётся |
| obs_bad_time | `25:99` | ❌ пропуск |
| obs_no_time | пустое время | ❌ пропуск |
| obs_suspicion_clamp | suspicion 99, цвет фиолетовая | ✅ подозр. → 10, окрас **неизвестен** |
| obs_night_ok | вторая отметка fox_010 ночью | ✅ |

**Итого после импорта:** 5 записей, 5 лис, 3 пропуска, несколько предупреждений в отчёте под формой.

---

## Бонус для ревьюера (не файл)

Вставить вручную битый JSON `{` — должна быть ошибка «Некорректный JSON» без замены данных.

Массив из 101 разных `fox_id` — отказ «Слишком много уникальных лис».
